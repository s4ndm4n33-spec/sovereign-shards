# End-to-End Test Build — Sovereign Shards Full System Validation

> **Purpose:** Verify every major subsystem in a single continuous session.
> Router, LLM tool loop, bug fixes, write/edit, defence suite, memory,
> context management, error recovery, agent mode — all 20 turns, no restart.
>
> **Prerequisite:** `git pull` to ensure all latest fixes are on disk.

---

## Test Protocol

1. Start a fresh J session: `python run.py`
2. Enter each prompt **exactly as written** (copy-paste)
3. After each response, record ✅ PASS / ❌ FAIL / ⚠ PARTIAL
4. Do **not** restart between turns — sustained context is the point
5. If J crashes, note which turn and restart from Turn 1

### Scoring

| Result | Meaning |
|--------|---------|
| 18-20 PASS | **SHIP IT** — full system validated |
| 15-17 PASS | Close — fix specific failures, re-run failed block |
| 10-14 PASS | Systemic issues — investigate before Phase 2 |
| < 10 PASS  | Fundamental regression — do not proceed |

*Pass:* No "You must call a tool" prompt. J's text answer is the final response.

```

```
*Expected:* J outputs `ACTION:{...}` → clean stop → real `[TOOL EXECUTION]` from the system.
**Watch:** There should be NO fake tool output generated by J between ACTION and the real result.
*Pass:* Only ONE `[TOOL EXECUTION]` block (the real one). No hallucinated results.

### Turn 9 — Bug 2 fix: bare ACTION fallback
```
how many lines are in app/chat.py?
```
*Expected:* J may call `run_bash wc -l app/chat.py` or similar. If J outputs a bare
`ACTION:run_bash wc -l app/chat.py` (no JSON), the fallback parser handles it.
*Pass:* J gets a real answer (line count). No "Invalid ACTION" errors.
*Note:* J might use JSON format (which is fine) — if so, this tests the standard path.

### Turn 10 — Error recovery: bad tool call

*Expected:* J calls `run_read this_does_not_exist_xyz.py` → file not found error.
Error nudge shows: `"Your last tool call FAILED: [Errno 2]..."`. J responds sensibly.
*Pass:* J acknowledges the file doesn't exist. No blind retry. No hallucinated file content.

## Block C: Write & Edit Pipeline (Turns 11-14)

### Turn 11 — Write a new file
```
create a file called test_e2e.py with this content: print("sovereign shard")
```
*Expected:* J calls `run_write test_e2e.py 'print("sovereign shard")'` or similar.
*Pass:* File created. J confirms.

### Turn 12 — Surgical edit with run_str_replace
```
use run_str_replace to change "sovereign shard" to "J was here" in test_e2e.py
```
*Expected:* J outputs:
```
ACTION:{"tool": "run_str_replace", "args": ["{\"path\": \"test_e2e.py\", \"old\": \"sovereign shard\", \"new\": \"J was here\"}"]}
```
Key: single JSON-string arg, not separate args.
*Pass:* Edit succeeds. J confirms the replacement.
*Partial:* J gets format wrong on first try, error nudge corrects it on retry.

### Turn 13 — Verify the edit
```
run_read test_e2e.py
```
*Expected:* Router intercepts → shows file with `print("J was here")`.
*Pass:* Content matches the edit from Turn 12. No leftover "sovereign shard".

### Turn 14 — Execute the edited file
```
python test_e2e.py
```
*Expected:* Router intercepts → `run_bash python test_e2e.py` → prints `J was here`.
*Pass:* Output is exactly `J was here`. Clean execution.
